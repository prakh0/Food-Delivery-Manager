# [Food Delivery Manager](https://food-delivery-manager.vercel.app/)

A React-based web application that manages food delivery orders and automatically assigns deliveries to the nearest eligible order.

This project was developed as part of the **Online Food Delivery Order Manager Assignment**.

# Objective

The goal of this application is to manage food delivery orders and automatically assign deliveries to the nearest available order while providing filtering, tracking, and activity monitoring capabilities.

---

# Features

## Order Management

Users can:

* Add new food delivery orders.
* View all existing orders.
* Delete orders.
* Mark delivered orders as paid.

Order IDs are automatically generated by the system using a unique format:

```text
#45231
#83914
#12567
```

This removes the need for manual ID entry and prevents duplicate order IDs.

---

## Order Status Workflow

Every order moves through the following states:

```text
Pending
    ↓
Out For Delivery
    ↓
Paid
```

### Pending

Order has been created but not assigned.

### Out For Delivery

Order has been assigned to a delivery agent.

### Paid

Order delivery has been completed and payment received.

Status badges are color-coded for better visibility.

---

## Dashboard

The application provides a dashboard that displays:

* Total Orders
* Pending Orders
* Orders Out For Delivery
* Paid Orders

The dashboard is displayed in a two-row responsive layout for improved readability.

---

## Search Orders

Users can search orders by Order ID.

Example:

```text
#45231
```

The table updates instantly to display matching results.

---

## Filter Orders

Orders can be filtered by:

### Status

* All
* Pending
* Out For Delivery
* Paid

### Maximum Delivery Distance

Orders can be filtered based on a maximum delivery distance.

The application enforces a maximum distance limit of:

```text
25 KM
```

---

## Delivery Assignment Logic

The application implements an automatic delivery assignment algorithm.

Rules:

1. Only unpaid orders can be assigned.
2. Orders already out for delivery are ignored.
3. Only orders within the selected distance range are considered.
4. The nearest eligible order is selected.
5. If no eligible order exists, the application displays:

```text
No order available
```

### Example

Orders:

```text
#101 → 12 KM
#102 → 4 KM
#103 → 8 KM
```

Assigned Order:

```text
#102
```

because it is the nearest eligible order.

---

## Activity Log

The Output Panel maintains a history of application events.

Examples:

```text
[4:12:32 PM] Order #45231 from KFC is Out For Delivery

[4:14:01 PM] Order #45231 marked as Paid

[4:15:22 PM] Order #45231 deleted
```

### Features

* Timestamped entries
* Most recent events displayed first
* Persists after page refresh
* Can be cleared by the user

---

## Export Orders

Users can export all orders as a JSON file.

Example:

```json
[
  {
    "orderId": "#45231",
    "restaurantName": "KFC",
    "itemCount": 2,
    "isPaid": false,
    "deliveryDistance": 5
  }
]
```

This can be used for backup or reporting purposes.

---

## Data Persistence

The application uses browser Local Storage.

Stored Data:

* Orders
* Activity Logs

Benefits:

* Data remains available after refreshing the page.
* No backend database required.
* Faster user experience.

---

## Responsive Design

The application is responsive across:

* Desktop
* Tablet
* Mobile Devices

The orders table is wrapped inside a responsive container that allows horizontal scrolling on smaller screens while maintaining layout integrity.

# Validation & Error Handling

The application validates all user inputs before submission.

### Validation Rules

#### Restaurant Name

* Required

#### Item Count

* Required
* Must be greater than 0

#### Delivery Distance

* Required
* Cannot be negative
* Cannot exceed 25 KM

#### Order IDs

* Automatically generated
* Guaranteed unique

### Error Handling

Examples:

```text
All fields are required
```

```text
Item count must be greater than 0
```

```text
Distance cannot be negative
```

```text
Maximum delivery distance is 25 KM
```

```text
No order available
```

# Running Locally

Clone the repository:

```bash
git clone <repository-url>
```

Move into the project:

```bash
cd food-delivery-manager
```

Install dependencies:

```bash
npm install
```

Start development server:

```bash
npm run dev
```

Build production bundle:

```bash
npm run build
```

Preview production build:

```bash
npm run preview
```
